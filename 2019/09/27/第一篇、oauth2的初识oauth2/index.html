第一篇、oauth2 [初识oauth2]

2019-09-27 java分类 评论

oauth2是开放授权的一个标准，旨在让用户允许第三方应用去访问改用户在某服务器中的特定私有资源，而可以不提供其在某服务器的账号密码给到第三方应用

可以将OAuth2分为四个角色：

  1. Resource Owner：资源所有者 即上述中的微信用户
  2. Resource Server：资源服务器 即上述中的微信服务器，提供微信用户基本信息给到第三方应用
  3. Client：第三方应用客户端 即上述中你公司正在开发的第三方应用
  4. Authorication Server：授权服务器 该角色可以理解为管理其余三者关系的中间层

不难看出，OAuht2 解决问题的关键在于使用授权服务器提供一个访问凭据给到第三方应用，让第三方应用可以在不知道资源所有者在资源服务器上的账号和密码的情况下，能获取到资源所有者在资源服务器上的受保护资源，这里的受保护资源就是微信用户的姓名以及头像等信息。
